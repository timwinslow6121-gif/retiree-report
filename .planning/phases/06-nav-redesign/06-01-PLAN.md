  <action>
In assets/css/main.css, find the `.site-nav__link` block (around line 419) and make these changes:

1. Change the transition property from `var(--transition-fast)` to `var(--transition-base)` so hover animations complete in 220ms (satisfies NAV-01):
   ```
   transition: color var(--transition-base), background var(--transition-base);
   ```

2. SPLIT the combined `.site-nav__link:hover, .site-nav__link[aria-current="page"]` rule into two separate rules:

   Hover rule (color + background tint only — no underline):
   ```css
   .site-nav__link:hover {
     color: var(--color-gold-light);
     background: rgba(255, 255, 255, 0.06);
   }
   ```

   Active/current rule (gold color + bold weight + gold underline — per user decision NAV-02):
   ```css
   .site-nav__link[aria-current="page"] {
     color: var(--color-gold-light);
     background: rgba(255, 255, 255, 0.06);
     font-weight: 900;
     border-bottom: 2px solid var(--color-gold-light);
   }
   ```

   Note: Use `border-bottom` not `text-decoration` for the underline — border-bottom gives exact thickness control and is visually cleaner for uppercase nav text. 2px thickness at the link's bottom edge is visually balanced.

Do NOT touch any other selectors in this file during this task.

In assets/css/main.css, find the `.site-nav__link` block updated in Task 1. Add `min-height: var(--tap-min)` and change `display: block` to `display: flex; align-items: center` so the 44px tap area is reliably achieved on mobile without requiring precise vertical tapping (satisfies NAV-03 and NAV-04).

The updated base rule should read:
```css
.site-nav__link {
  display: flex;
  align-items: center;
  min-height: var(--tap-min);
  color: #C5D3E0;
  text-decoration: none;
  font-family: var(--font-sans);
  font-size: var(--text-sm);
  font-weight: 700;
  letter-spacing: 0.5px;
  text-transform: uppercase;
  padding: var(--space-2) var(--space-3);
  border-radius: var(--radius-sm);
  transition: color var(--transition-base), background var(--transition-base);
}
```

On desktop (min-width: 768px media query), the link height is naturally constrained by the horizontal nav bar — the min-height will be effectively overridden by the flex row alignment. This is acceptable; 44px min-height on desktop nav links does no harm.

Do NOT add any media-query scoping for the min-height — applying it universally is simpler and correct.
