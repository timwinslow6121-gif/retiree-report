Page front matter contract (required for every migrated page):
```yaml
---
layout: layouts/page.njk
title: [Page-specific title]
description: [Meta description]
permalink: /pages/[filename].html   # (or /index.html for homepage)
last_updated: [Date string, e.g. "March 2026"]  # Omit if page never had one
---
```

Boilerplate to STRIP from legacy HTML content (owned by base.njk/page.njk — do not duplicate):
- Entire `<html>`, `<head>`, `<body>` tags
- `<a href="#main-content" class="skip-nav">` element
- `<div class="banner" ...>` banner block
- Entire `<header class="site-header">` block including nav
- `<footer class="site-footer">` block
- Inline `<script>` nav toggle block
- `<main id="main-content">` wrapper tags (opening and closing)
- `</body></html>` closing tags

What to KEEP: everything inside `<main id="main-content">...</main>` — that is the page-specific content.

Relative paths: any `../assets/` or `../` references in the kept content must be changed to `/assets/` or `/`.
