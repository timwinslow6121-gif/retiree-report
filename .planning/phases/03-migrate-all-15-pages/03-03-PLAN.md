  <action>
    Read `assets/css/main.css` and audit for duplicate CSS blocks. This is a high-priority task per the CONTEXT.md decisions — deduplication happens during migration, not after.

    **What to look for and remove:**
    - Duplicate reset rules (e.g., `* { box-sizing: border-box }` appearing more than once)
    - Duplicate component class definitions (e.g., `.btn {}` defined in two separate locations)
    - Duplicate custom property declarations (`--color-*`, `--font-*` custom properties declared twice)
    - Exact or near-exact duplicate selector blocks (same selector, same properties, different location)

    **How to deduplicate:**
    1. Identify the first occurrence of each duplicated block — that is the canonical one
    2. Remove subsequent duplicate occurrences entirely
    3. If two blocks have the same selector but different properties (legitimate overrides for different contexts), leave both — only remove true duplicates

    **What NOT to change:**
    - Media query blocks (they are context-dependent, not duplicates even if selector matches)
    - Intentional specificity overrides (`.page-content .btn {}` is not a duplicate of `.btn {}`)
    - Any rule that affects visual appearance where you are unsure — if in doubt, leave it
